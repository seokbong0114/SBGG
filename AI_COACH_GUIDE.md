# 🧠 AI 인게임 코치 — 프롬프트 엔지니어링 & 운영 가이드

SB.GG의 킬러 기능 **초개인화 인게임 코칭 리포트**의 설계·프롬프트·운영 문서.
목표: "존재하지 않는 메타/아이템을 지어내는 환각 없이, 실제 게임 수치에만 근거해
**핵심 문제 1개 + 실행 가능한 개선책 1개**를 제시하는 챌린저 코치".

---

## 1. 데이터 흐름 (파이프라인)

```
전적 카드 [✨ AI 리포트] 클릭
   └─ (JS) 실제 매치 요약 수치를 POST  → /api/ai_coach
        └─ generate_mock_timeline()   ← 실제 K/D/A·CS·킬관여를 시드로 타임라인 구성
             └─ build_coach_prompt()  ← System + User 프롬프트 조립
                  └─ call_llm_coach() ← LLM 호출 or 데모 폴백
                       └─ JSON 리포트 → 캐시 저장 → 프론트 카드 렌더
```

> **실측 전환 지점은 단 한 곳:** `generate_mock_timeline()` 을 match-v5 timeline
> 파서로 교체하면, 프롬프트·LLM·UI·캐시는 그대로 두고 실측 코칭으로 전환된다.

---

## 2. 환각(Hallucination) 방지 3중 방어

| 계층 | 방어 수단 |
|---|---|
| **프롬프트** | "제공된 <MATCH_DATA> 수치에만 근거", "없는 아이템·룬·챔피언·패치 언급 금지", "확신 없으면 범주명(코어/방어 아이템)으로만" |
| **출력 구조** | JSON 스키마 강제 → 자유서술로 흘러가 상상하는 것을 원천 차단. `max_tokens=700`으로 장황한 창작 억제 |
| **파싱** | `re.search(r'\{.*\}')` 방어적 JSON 추출 → 실패 시 데모 리포트로 폴백(빈 화면/깨진 출력 없음) |

---

## 3. System Prompt (핵심 제약)

`COACH_SYSTEM_PROMPT` (app.py). 규칙을 **번호로 명시**해 모델이 각 항목을 체크리스트처럼 지키게 함:

1. 오직 `<MATCH_DATA>` 수치에만 근거 — 없는 사실 창작 금지
2. 없는 아이템·룬·챔피언·패치 언급 금지 (확신 없으면 범주명만)
3. 핵심 문제점 **정확히 1개** (가장 뚜렷한 약점 선택)
4. 실행 가능한 개선책 **1개** (다음 게임 즉시 적용 지침)
5. 근거 없는 칭찬·일반론 금지 → **반드시 수치 인용**
6. 한국어, 친근하지만 전문적인 코치 말투
7. **JSON 스키마로만** 응답 (그 외 텍스트·마크다운·코드펜스 금지)

> **왜 "1개"로 제한?** 초보자에게 5가지를 던지면 아무것도 못 고친다.
> 한 게임에서 가장 임팩트 큰 문제 하나에 집중시키는 것이 실제 실력 향상에 효과적.

## 4. User Prompt

```
다음은 분석 대상 1게임의 데이터다.
<MATCH_DATA>
{ ...타임라인 JSON... }
</MATCH_DATA>
위 데이터만 근거로, 규칙에 따라 JSON 코칭 리포트를 작성하라.
```

- 데이터는 `<MATCH_DATA>` 태그로 감싸 **경계를 명확히** (프롬프트 인젝션·혼동 방지)
- 지표: KDA, 킬관여%, 분당CS, CS@10, 15분 골드차, 분당시야, 제어와드, 14분내 데스 등

## 5. 강제 출력 스키마

```json
{
  "headline": "한 줄 총평(25자 내외)",
  "problem":  { "title": "핵심 문제 제목", "detail": "수치 인용 2~3문장" },
  "solution": { "title": "개선책 제목",   "detail": "다음 게임 실행 지침 2~3문장" },
  "stat_highlight": "가장 문제되는 핵심 수치 하나(예: 14분 내 데스 4회)"
}
```

프론트(`renderCoachCard`)가 이 스키마를 그대로 카드로 렌더 → 백엔드/프론트 계약 고정.

## 6. 활성화 방법 (환경변수)

| 변수 | 기본값 | 설명 |
|---|---|---|
| `AI_COACH_ENABLED` | `1` | 전적 카드에 버튼 노출 (0=완전 숨김) |
| `AI_COACH_LIVE` | `0` | **0 = 데모(무료·키불필요), 1 = 실제 LLM 호출** |
| `AI_COACH_MODEL` | `claude-sonnet-5` | 코치 모델 |
| `ANTHROPIC_API_KEY` | (없음) | LIVE=1 시 필수 |

- **지금(데모 모드)**: 키 없이도 실제 게임 수치 기반 리포트가 나온다. UI/UX·캐시·프론트 계약을 100% 검증 가능. 카드에 "데모 · 예시 분석" 배지 표시.
- **실제 LLM 전환**: `pip install anthropic` → Render 환경변수에 `ANTHROPIC_API_KEY` + `AI_COACH_LIVE=1` 설정. 배지가 "AI 실측"으로 바뀐다.

### 모델 선택 (품질 vs 비용)
- `claude-haiku-4-5` — 가장 저렴·빠름. 트래픽 많고 예산 민감할 때.
- `claude-sonnet-5` — **권장 기본값**. 코칭 품질과 비용 균형.
- `claude-opus-4-8` — 최고 품질. 프리미엄 분석/유료 기능화 시.

### 비용 방어
- 리포트는 매치 시그니처(챔피언+K/D/A+CS+킬관여) 해시로 **14일 캐시**(`aicoach#...`). 같은 게임 재클릭 시 LLM 재호출 0.
- `max_tokens=700` 상한으로 응답당 토큰 고정.

---

## 7. 신뢰 원칙 준수 (중요)

SB.GG 원칙: **신뢰할 수 없는 정보는 절대 노출하지 않는다.**

- 데모 리포트는 **합성 타임스탬프가 아닌 실제 집계 수치**(데스·분당CS·킬관여·KDA·승패)에만 근거 → 환각 0.
- 카드 하단에 "실제 게임 수치 기반 예시 · 실측 연동 시 초 단위 분석 전환" 문구로 **솔직하게 고지**.
- 초 단위 실측(누가 언제 죽었는지)은 프로덕션 키·타임라인 연동 후에만 "AI 실측" 배지로 제공.

---

## 8. 향후 고도화 로드맵

1. **실측 타임라인 연동** — `generate_mock_timeline()` → match-v5 `/timeline` 파서 교체 (개인 키로도 과거 경기 타임라인 조회 가능).
2. **상대 라이너 대비 분석** — 같은 라인 상대와의 CS/골드/데스 격차를 프롬프트에 주입.
3. **티어 벤치마크 결합** — 이미 수집 중인 `tier_benchmark`(에메랄드+ 평균)를 프롬프트에 넣어 "네 티어 평균 대비" 코칭.
4. **연속 경기 패턴** — 최근 N게임을 묶어 "반복되는 약점" 진단(단발 게임 노이즈 제거).
